# Content Bible

本文件用来固定《山巅微光》的内容方向，避免后续功能越堆越散。它不是剧情大纲的最终稿，而是给设计、代码、测试和 AI 协作使用的统一标准。

## 一句话定位

《山巅微光》是一个以冲刺、攀墙、光继点和路线练习为核心的精密平台跳跃 vertical slice。玩家不是单纯通关，而是在每个房间里理解路线、压缩失误、追踪 PB、完成 Drill 合同，并通过复盘系统找到下一步训练目标。

## 核心体验

- **短房间，高反馈**：每个房间都应该在几十秒内完成，并立刻给出 split、PB、clean、Flow 或 Drill 反馈。
- **可读路线优先**：难度来自路线选择和动作节奏，不来自看不清、误导 UI 或不可预测判定。
- **训练不是菜单负担**：Practice 面板服务于复盘和定位薄弱房，而不是打断普通游玩。
- **普通游玩安静**：普通游玩时 HUD 克制；训练和复盘中才展示更多指标。
- **机制有边界**：光继点、风升流、超载棱镜、回声锚点、脆冰都应服务于路线节奏，不做无限悬空或无意义特效。

## 当前 vertical slice 范围

当前公开 demo 是 10 房纵向切片，不是完整章节。

| 房间段 | 目标 |
| --- | --- |
| R1-R3 | 建立跳跃、冲刺、落点、弹簧和基础节奏 |
| R4-R6 | 组合光继点、路线选择、恢复和出口节拍 |
| R7-R8 | 引入后段压力：风、脆冰、棱镜、移动端可读性 |
| R9-R10 | 用回声、棱镜、光继与终盘耐压完成收束 |

## 机制角色

### 跳跃 / 冲刺 / 攀墙

基础动作必须保持可靠、可预测、可练习。任何手感改动都要说明是否影响：

- coyote time
- jump buffer
- dash buffer
- dash aim memory
- wall coyote
- wall jump lock
- spark hop window

### 光继点

光继点是受限续航机制。它只奖励冲刺、高速或续跃状态，不应变成无限空中续航。连续触发可以有连锁反馈，但要保持路线意图清楚。

### 风升流

风升流改变空中节奏，应该让玩家提前规划高度和落点。视觉上必须有边界和上升方向提示，不能只靠试错理解。

### 超载棱镜

棱镜是高速路线的上限工具。它应该制造“安全线 vs 快线”的选择，而不是强制所有玩家只走高手线。

### 回声锚点

回声减少练习摩擦，帮助玩家在复杂房间恢复节奏。它不是撤销所有错误的保险，而是让路线练习更快重试。

### 脆冰

脆冰用于后段节奏压力。玩家踩上后应该能通过视觉进度和危险描边理解“立刻离开”。不要在前 6 房引入脆冰。

## 训练内容标准

每个房间至少应该拥有：

- `target`：目标时间。
- `safe route`：稳定通关路线。
- `fast route`：更快但更难的路线。
- `expert route`：高手线，必须对应关键动作。
- `Clean` 合同：无失误完成。
- `Pace` 合同：达到 target 或目标节奏。
- `Style` 合同：房间专属动作挑战。
- `Expert` 合同：速度 + 无失误 + 关键动作验证。
- `Focus` 指标：死亡原因、失败类型、最大 split 损失、下一 Drill。

## 文案语气

文案应短、克制、像训练提示，不像教程弹窗。

推荐：

- “先稳落点，再追速度。”
- “本房建议：Clean R4，再练 Pace。”
- “最大损失来自 R7：风场后落点过低。”

避免：

- 长篇解释机制。
- 在普通游玩中持续弹出教学卡。
- 用胜负/惩罚语气评价玩家。

## 内容扩展规则

在以下条件满足前，不增加 R11+：

1. 至少完成一次 R1-R10 真人完整通关。
2. 记录每房死亡原因、看不清点、卡顿点、误触点。
3. `KNOWN_ISSUES.md` 已更新真实限制。
4. 至少跑过 `npm run check`。
5. 如果改动涉及触控、设置、练习、存档或输入，跑过 `npm run browser-smoke` 或完成等价人工覆盖。

## 后续内容方向

优先顺序：

1. 打磨 R1-R10 的可读性、难度曲线和复盘质量。
2. 补强训练文案，让每房“为什么练这一房”更清楚。
3. 增加路线 replay/ghost 的可解释性。
4. 完成第一章主题和视觉统一。
5. 再考虑第二组房间或新机制。

## 不扩展的内容

- 不做账号、排行榜、云存档。
- 不做第二套成就系统。
- 不做剧情长文本压过练习体验。
- 不为炫技加入无法被 `check-maps` 或人工清单验证的复杂机制。
