# Playtest Protocol

本文件用于把人工试玩从“随便玩一下”变成可复盘、可对比、可转成 Issue 的流程。脚本可以检查地图、状态、UI 和存档，但真实手感、可读性、难度曲线和设备体验必须靠人工试玩确认。

## 试玩原则

- 先记录问题，再考虑解决方案。
- 优先记录真实卡点，不记录“也许以后可以”的愿望。
- 每个问题尽量附带房间号、操作、设备、视口、是否使用手柄/触控。
- 普通游玩和练习模式分开测，不混在同一条反馈里。

## 准备

1. 打开线上版本或本地版本。
2. 记录版本号：HTML `meta build-version` 或启动界面显示的 build。
3. 清空或备份 localStorage。
4. 设备信息：桌面键盘 / 手柄 / 手机触控 / 平板触控。
5. 视口信息：桌面尺寸，或移动端竖屏/横屏。

## 3 分钟首玩测试

目标：判断第一次打开是否知道能做什么。

记录：

- 是否能立即找到开始按钮。
- 是否知道移动、跳跃、冲刺、抓墙。
- 第一次死亡发生在哪里。
- 第一次卡住超过 20 秒的房间。
- 是否被设置/练习按钮干扰。
- HUD 是否太吵或太少。

通过标准：玩家无需读 README，也能开始移动、跳跃、冲刺并通过 R1 或理解正在练习什么。

## Practice 面板测试

测试路径：

1. 打开练习面板。
2. 选择任意房间。
3. 进入 Clean Drill。
4. 失败一次，确认是否重开当前 Drill。
5. 进入 Pace Drill。
6. 中断 Route 或 Feel Lab，再恢复。
7. 查看练习报告和房间掌握表。

通过标准：Practice 面板回答三个问题：

- 我现在该练哪一房？
- 为什么练这一房？
- 用哪个 Drill 练？

## 设置与反馈测试

必须覆盖：

- `O` 打开/关闭设置。
- `Escape` 关闭面板。
- 设置面板里输入反馈备注时，按 `O/P/Escape/R/T` 不误触发游戏。
- 复制诊断。
- 复制反馈模板。
- 导出存档。
- 粘贴错误 JSON，确认不刷新。
- 导入有效存档，确认写入备份。
- 恢复导入前备份。

## 移动端测试

至少覆盖两个视口：

- 390x700 竖屏。
- 700x390 横屏。

记录：

- 是否出现横向滚动。
- 触控按钮是否遮住关键平台。
- 设置面板是否裁切。
- Room 下拉框和 Drill 按钮是否可点。
- 手指是否容易误按。
- 浏览器地址栏出现/隐藏时坐标是否错位。

## 手柄测试

记录：

- 浏览器是否检测到手柄。
- deadzone 默认值是否能压住漂移。
- A/B/X/RT/肩键是否符合直觉。
- 长时间抓墙是否疲劳。
- 是否需要不同预设。

## 分级

- P0：无法开始、无法部署、页面空白、存档损坏、关键输入失效。
- P1：阻止 R1-R10 正常通关或严重影响移动端/手柄。
- P2：明显摩擦、路线误导、复盘建议不清楚。
- P3：文案、视觉、轻微体验优化。

## 试玩后动作

1. 把真实问题写入 Issue。
2. 如果是已知但暂不修，更新 `KNOWN_ISSUES.md`。
3. 如果影响发布，更新 `RELEASE_CHECKLIST.md`。
4. 如果改变内容方向，更新 `CONTENT_BIBLE.md`。
